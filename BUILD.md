# Android Build Guide — RoboPet

This guide explains how to build a **single Android APK** that combines the Ionic/Angular frontend and the Python (FastAPI + Socket.IO) backend. The Python backend runs as a local in-process service on the device; the Ionic WebView connects to it on `localhost`.

---

## Architecture Overview

```
┌─────────────────────────────────────────────────┐
│                  Android APK                     │
│                                                  │
│  ┌─────────────────────────┐   localhost:8000       │
│  │  Ionic WebView           │ ◄──────────────────►  │
│  │  (Angular app)           │                        │
│  │  @xenova/transformers    │  ┌──────────────────┐  │
│  │  Whisper ONNX (on-device)│  │  Python service  │  │
│  └─────────────────────────┘  │  FastAPI+Uvicorn  │  │
│                                │  Socket.IO       │  │
│  Capacitor plugins:            │  MediaPipe (stub)│  │
│  - Camera Preview              └──────────────────┘  │
│  - Voice Recorder                                     │
│  - Text-to-Speech                                     │
└─────────────────────────────────────────────────┘
```

The Python runtime is embedded using **[Chaquopy](https://chaquo.com/chaquopy/)** — a Gradle plugin that packages a CPython interpreter and pip packages directly into the Android project generated by Capacitor.

## Step 1 — Clone and install dependencies

```bash
git clone <repo-url> robopet
cd robopet

# Install Node/Ionic dependencies
cd mobile
npm install
cd ..
```

---

## Step 2 — Configure the backend environment

The Python backend reads configuration from `backend/.env`:

```bash
cp backend/.env.example backend/.env
```

Edit `backend/.env` and fill in your values:

```dotenv
OPENAI_API_KEY=sk-...
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini
WHISPER_MODEL=base          # base | small | medium
YOLO_MODEL=yolov8n.pt
ROBOT_NAME=RoboPet
LANGUAGE=en
HOST=0.0.0.0
PORT=8000
LOG_LEVEL=info
```

> The `.env` file will be bundled inside the APK (see Step 5). Do **not** commit it to git.

---

## Step 3 — Build the Ionic/Angular web assets

```bash
cd mobile
npm run build -- --configuration production
```

This produces the compiled web assets in `mobile/www/`.  
Verify the output:

```bash
ls mobile/www/
# Should contain: index.html, main.js, polyfills.js, styles.css, assets/, ...
```

---

## Step 4 — Add the Android Capacitor platform

```bash
cd mobile

# Add the android platform (only needed once per checkout)
npx cap add android

# Sync web assets and plugins into the android/ project
npx cap sync android
```

This creates `mobile/android/` — a standard Android Gradle project.

## Step 5 — Integrate the Python backend with Chaquopy

### 5.1 — Add Chaquopy to project-level `build.gradle`

Edit `mobile/android/build.gradle`:

```groovy
buildscript {
    repositories {
        google()
        mavenCentral()
        maven { url "https://chaquo.com/maven" }   // add this line
    }
    dependencies {
        classpath 'com.android.tools.build:gradle:8.4.0'
        classpath "com.chaquo.python:gradle:16.0.0"  // add this line
    }
}
```

> Check [Chaquopy releases](https://github.com/chaquo/chaquopy/releases) for the latest version compatible with your AGP version.

### 5.2 — Apply Chaquopy to the app module

Edit `mobile/android/app/build.gradle`. Add Chaquopy below the existing `apply plugin` lines and configure it inside `android {}`:

```groovy
apply plugin: 'com.android.application'
apply plugin: 'com.chaquo.python'    // add this line

android {
    // ... existing config ...

    defaultConfig {
        // ... existing config ...

        ndk {
            // ABIs to support — include all you need
            abiFilters "arm64-v8a", "x86_64"
        }

        python {
            // Match the Python version in your requirements.txt
            version "3.11"

            pip {
                // FastAPI stack
                install "fastapi>=0.115,<1"
                install "uvicorn[standard]>=0.32,<1"
                install "python-socketio>=5.11,<6"
                install "pydantic>=2.9,<3"
                install "pydantic-settings>=2.6,<3"
                install "python-dotenv>=1.0,<2"

                // OpenAI SDK — used for both chat AND Whisper STT API
                install "openai>=1.55,<2"

                // Vision helpers available on Chaquopy's mirror
                install "numpy>=1.26,<3"
                install "opencv-python-headless>=4.10,<5"

                // NOTE: faster-whisper and mediapipe are NOT installed here.
                // See the note below for the STT strategy used on Android.
            }
        }
    }
}
```

> **STT on Android — on-device Whisper via `@xenova/transformers` in the Ionic WebView**
>
> `faster-whisper` requires `ctranslate2`, a native C++ library. No Android-compatible wheel exists on Chaquopy's PyPI mirror, so it cannot be embedded in the APK.
>
> **The solution applied:** STT has been moved entirely to the Ionic/Angular frontend using [`@xenova/transformers`](https://github.com/xenova/transformers.js) — the WebAssembly/ONNX port of Hugging Face Transformers. `WhisperService` (`src/app/services/whisper.service.ts`) lazy-loads the `Xenova/whisper-base` ONNX model into IndexedDB on first use and runs inference in the WebView. After the initial download (~150 MB) the app works fully offline.
>
> The new voice flow:
> 1. User holds the mic button → `capacitor-voice-recorder` captures audio.
> 2. On release → `WhisperService.transcribe()` decodes the audio via the Web Audio API, resamples to 16 kHz, and runs Whisper inference locally.
> 3. The resulting text is sent as a `chat_message` WebSocket event (no `audio_data` event is used anymore).
> 4. The Python backend (`socket_handlers.py`) processes `chat_message` as before → LLM → `robot_response`.
>
> The button shows a **yellow `cloud-download-outline` icon** while the model is being downloaded, and a **yellow `hourglass-outline` icon** while transcription is running. After that it behaves as before (red `mic` while recording, blue `mic-outline` at idle).
>
> `mediapipe` is excluded for the same reason (no Android wheel). `VisionService` is already a stub in the codebase and will simply return empty results until a future implementation is added.

### 5.3 — Copy the Python source code into the Android Python source root

Chaquopy looks for Python code under `android/app/src/main/python/`. Create the directory and copy the backend:

```bash
mkdir -p mobile/android/app/src/main/python
cp -r backend/app mobile/android/app/src/main/python/app
cp backend/.env mobile/android/app/src/main/python/.env
```

> Repeat the `cp` steps (or automate with a Gradle task) every time you change the backend code.

### 5.4 — Add a Python entry-point launcher

Create `mobile/android/app/src/main/python/start_server.py`:

```python
"""Launched by the Android Activity to start the FastAPI/Uvicorn server."""
import os
import threading

def run():
    # Point dotenv at the bundled .env
    base = os.path.dirname(__file__)
    os.environ.setdefault("ENV_FILE", os.path.join(base, ".env"))

    import uvicorn
    from app.main import app as asgi_app

    uvicorn.run(asgi_app, host="127.0.0.1", port=8000, log_level="info")

def start():
    t = threading.Thread(target=run, daemon=True)
    t.start()
```

### 5.5 — Start the Python server from the Android Activity

Edit `mobile/android/app/src/main/java/com/robopet/app/MainActivity.java`
(or the Kotlin equivalent). Chaquopy provides a `Python` singleton:

```java
import com.chaquo.python.Python;
import com.chaquo.python.android.AndroidPlatform;

public class MainActivity extends BridgeActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        // Start Python interpreter and server before the WebView loads
        if (!Python.isStarted()) {
            Python.start(new AndroidPlatform(this));
        }
        Python py = Python.getInstance();
        py.getModule("start_server").callAttr("start");

        super.onCreate(savedInstanceState);
    }
}
```

> The `BridgeActivity` is the Capacitor base class. The Python server is started in a daemon thread so it does not block the UI.

### 5.6 — Point the Ionic app at localhost

Edit `mobile/capacitor.config.ts` so the WebView reaches the local server:

```typescript
import type { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'com.robopet.app',
  appName: 'RoboPet',
  webDir: 'www',
  server: {
    cleartext: true,              // allow plain HTTP to localhost
  },
};

export default config;
```

In `mobile/src/environments/environment.prod.ts` set the backend URL:

```typescript
export const environment = {
  production: true,
  backendUrl: 'http://127.0.0.1:8000',
};
```

Run `npx cap sync android` again to apply the config change.

## Step 6 — Add required Android permissions

Edit `mobile/android/app/src/main/AndroidManifest.xml` and ensure these permissions are present:

```xml
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.CAMERA" />
<uses-permission android:name="android.permission.RECORD_AUDIO" />
<uses-permission android:name="android.permission.MODIFY_AUDIO_SETTINGS" />

<!-- Required for cleartext localhost traffic on API 28+ -->
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />
```

Also add to the `<application>` tag:

```xml
<application
    android:usesCleartextTraffic="true"
    ...>
```

## Step 7 — Build the APK

### Debug build (for testing)

```bash
cd mobile/android
./gradlew assembleDebug
```

The APK is output to:

```
mobile/android/app/build/outputs/apk/debug/app-debug.apk
```

### Release build (for distribution)

1. Generate a signing keystore (one-time):
   ```bash
   keytool -genkey -v \
     -keystore robopet-release.jks \
     -alias robopet \
     -keyalg RSA -keysize 2048 \
     -validity 10000
   ```

2. Add signing config to `mobile/android/app/build.gradle`:
   ```groovy
   android {
       signingConfigs {
           release {
               storeFile file("../robopet-release.jks")
               storePassword System.getenv("KEYSTORE_PASSWORD")
               keyAlias "robopet"
               keyPassword System.getenv("KEY_PASSWORD")
           }
       }
       buildTypes {
           release {
               signingConfig signingConfigs.release
               minifyEnabled false
           }
       }
   }
   ```

3. Build:
   ```bash
   cd mobile/android
   KEYSTORE_PASSWORD=<your-pass> KEY_PASSWORD=<your-pass> \
     ./gradlew assembleRelease
   ```

   Output: `mobile/android/app/build/outputs/apk/release/app-release.apk`

## Step 8 — Install and run on a device

Connect your Android device with **USB debugging** enabled, or start an emulator (x86_64 AVD).

```bash
# Check connected devices
adb devices

# Install the debug APK
adb install mobile/android/app/build/outputs/apk/debug/app-debug.apk

# Stream logs (filter to RoboPet and Python)
adb logcat -s "Capacitor" "python.stdout" "python.stderr"
```

You can also open the project in Android Studio and use **Run ▶** to build, deploy, and debug in one step:

```bash
npx cap open android
```

## Quick Reference — Full Build Sequence

```bash
# 1. Build web assets
cd mobile && npm run build -- --configuration production && cd ..

# 2. Copy Python backend into Android source tree

# 3. Sync Capacitor
cd mobile && npx cap sync android && cd ..

# 4. Build APK
cd mobile/android && ./gradlew assembleDebug && cd ../..

# 5. Deploy
adb install mobile/android/app/build/outputs/apk/debug/app-debug.apk
```

## Troubleshooting

| Problem | Likely cause | Fix |
|---------|-------------|-----|
| `No matching distribution found for mediapipe` | No Android wheel in Chaquopy mirror | Remove from pip block; guard `import mediapipe` with `try/except ImportError` |
| `No matching distribution found for faster-whisper` | `ctranslate2` not available for Android | Remove from pip block; guard `from faster_whisper import ...` with `try/except ImportError` |
| `CLEARTEXT communication not permitted` | Missing `usesCleartextTraffic` | Add the flag to `AndroidManifest.xml` |
| WebView shows blank page | Web assets not synced | Re-run `npx cap sync android` |
| Python server not starting | Activity init order | Ensure `Python.start(...)` is called before `super.onCreate()` |
| `java.lang.UnsatisfiedLinkError` | ABI mismatch | Confirm `abiFilters` includes the device ABI |
| `NDK not configured` | Chaquopy requires NDK | Install NDK 26.x via Android Studio SDK Manager |
